# Republic with No Masters

Use this skill to explain, summarize, critique, or adapt the framework using principles the user provides in the conversation.

## Working method
- First identify whether the request is summary, explanation, critique, FAQ, or adaptation.
- Preserve the framework's own terms when the user supplies them.
- Separate faithful restatement from your own interpretation or extension.
- If key principles are missing, ask for them rather than inventing doctrine.

## Output options
- Short summary for newcomers.
- Structured overview of core mechanisms and constraints.
- FAQ, critique, or implementation sketch.
- Public-facing manifesto-like prose when requested.

## Guardrails
- Do not claim real-world adoption or readiness without evidence.
- Mark speculative proposals clearly.
- Keep the language precise and internally consistent.
